Framework 02

Meta Ads Rapid Scaling Framework

A disciplined Test → Learn → Scale sequence for moving brands from audience discovery to full-funnel performance on Facebook & Instagram.

Stage 1
Testing Phase
20% of Budget
  • 5–7 interest-based ad sets targeting distinct audience clusters
  • Campaign objective: Purchase or Lead — never Engagement
  • Equal budget per ad set; identical creative to isolate signal
  • Minimum threshold: 50 conversions or 7 days before evaluation
  • Monitor CPM, CTR, CPC, Cost per Purchase, ROAS
Stage 2
Learning Phase
30% of Budget
  • Build Lookalike Audiences from: Purchasers → Website Buyers → Add-to-Cart
  • Test LAL 1%, 2%, 3% as separate ad sets — never combine
  • Introduce creative testing: 3 headline variants, 2 visual formats
  • Check audience overlap — consolidate if >60% overlap
  • Exit criteria: CPA within 20% of target for 7 consecutive days
Stage 3
Scaling Phase
50% of Budget
  • Horizontal scaling: duplicate winners into new campaigns
  • Vertical scaling: increase budget max 20–30% every 48 hrs
  • Add Broad Targeting sets — let Meta's algorithm self-optimise
  • Retargeting layer: 7/30/60-day visitors with dynamic creative
  • Refresh creative every 14–21 days; watch for frequency >3.0

Framework 03

Account Audit Template

The methodology I use to find wasted spend and strategy mismatches within the first 72 hours of accessing an account.

Search Term Audit
1
Pull 90-day search term report filtered by impressions >10, all match types
2
Segment by outcome: Converters vs Non-Converters vs Zero-impression clicks
3
Flag informational queries: "how to", "free", "what is" — high CTR, zero purchase intent
4
Identify geographic mismatches, brand contamination, and broad match drift
5
Detect duplicate coverage causing internal auction competition and inflated CPC
6
Build tiered negative keyword lists: Account → Campaign → Ad Group level
Bidding Strategy Review
1
<30 conv/month → Maximise Clicks with bid cap — insufficient data for Smart Bidding
2
30–50 conv/month → Maximise Conversions, no target — let algorithm learn first
3
50+ conv/month → tCPA or tROAS — statistical significance achieved
4
Flag tCPA/tROAS targets set below historical achievable averages
5
Identify bid adjustments stacked on Smart Bidding — creates algorithm noise
6
Review portfolio bid strategies that group dissimilar campaigns together

Framework 04

The Technical Edge

How a Computer Science engineering degree enables deeper platform thinking — beyond what most platform-trained marketers can offer.

Conversion Tracking Accuracy

I debug JavaScript event listeners and GTM tag firing directly in DevTools. Reading dataLayer objects confirms transaction data is passing correctly. This means tracking errors that silently misdirect Smart Bidding spend get caught and fixed fast.

GA4 Data Interpretation

Understanding session vs event vs user scopes prevents misattribution errors. I use BigQuery exports for raw event-level LTV analysis and build custom GA4 Explorations that surface drop-off points invisible in standard reports.

Algorithmic Bidding Logic

Understanding how ML algorithms process signals means knowing that contradictory inputs create noise — not efficiency. I know exactly when Smart Bidding has enough data to be trusted and when it needs structured guidance to perform.

Structured Experimentation

Every A/B test has a clear hypothesis, one controlled variable, and a pre-calculated sample size for 95% statistical confidence — not just "run until it looks better." Every result is documented to build compounding knowledge over time.
